# Model estimation code map (2026-07)

This note is the entry point for the estimation code in this folder. Paths are
resolved by `config.py`; programs should not change the working directory.

## One source of truth for the loan specification

`budget_shock.py` is the shared interface for the budget-shock and borrowing-
preference specification. It owns the optimizer-vector layout, saved-file
schema, conditional shock mean and standard deviation, forward random draws,
backward quadrature, risk-aversion mapping, and debt-penalty mapping.

The three consumers are:

- `model_fitloans_dynamic.py`: unpacks trial SMM parameters and evaluates loan
  choices using the shared mappings;
- `model_solution_em.py`: loads the saved specification and uses its quadrature,
  risk aversion, and debt penalty in the backward solution;
- `model_simulation_em.py`: draws the same additive dollar shock and uses the
  same risk aversion and debt penalty in forward student borrowing.

To experiment with a different distribution or group parameterization, change
`budget_shock.py` and its parameter-count constants first. Do not recreate those
formulas separately in the estimator, solution, or simulation.

## Education-specific grant process

`financial_process.py` is the shared grant interface used by
`model_solution_em.py` and `model_simulation_em.py`. It loads separate receipt
logits and positive log-amount equations for two-year, four-year, and graduate
enrollment, applies the lognormal mean correction when saved residual sigmas
are available, and exposes matched scalar and vectorized expected-grant
functions. The backward solver uses the scalar function; forward simulation
uses the vectorized function. No unobserved financial type enters either one.

The required files are generated by `1_datacleaning/Clear NSL97/
data_model_fullfields.do`:

- `dummy_grants_educ1.npy`, `dummy_grants_educ2.npy`,
  `dummy_grants_grad.npy`;
- `paramgrants_educ1.npy`, `paramgrants_educ2.npy`,
  `paramgrants_grad.npy`;
- `sigma_grants_educ1.npy`, `sigma_grants_educ2.npy`,
  `sigma_grants_grad.npy`.

Rerun that Stata file before the next auxiliary EM estimation. Besides
refreshing the coefficient files, it copies `grants_grad` into the common
`totgrants` column for graduate-enrollment observations before exporting the
Python income panels. Existing coefficient files remain readable through a
compatibility mapping, but existing income panels do not contain the graduate
grant correction and therefore are not sufficient for the new EM grant block.

The auxiliary estimator in `model_em_algorithm.py` estimates the same three
education-specific grant hurdle models, a pooled parental-transfer hurdle
model, and three education-specific annual-loan hurdle models. Its auxiliary
latent distribution has sixteen joint classes: binary schooling type x binary
grant type x binary parental-transfer type x binary loan type. Loan receipt and
positive log amount depend on demographics plus realized grants and transfers;
separate baseline equations are used for two-year, four-year, and graduate
enrollment. Only enrolled person-years enter the loan equations. Expected
consumption remains cached for the four grant/transfer combinations, so the
loan type is currently identified by the loan measurement equations rather than
entering the 604-parameter choice block directly.

For this intermediate stage, `latent_types.py` deliberately remains the shared
eight-type structural interface. The auxiliary checkpoint/results contain the
full sixteen-column posterior and also save an explicit posterior collapsed over
the loan dimension as `auxiliary_q_eight_types.npy`. Structural estimation and
simulation should not consume the full sixteen-type result until their separate
loan-type conversion is implemented.

The active auxiliary choice objective keeps the exact closed-form score but
evaluates all individual-by-type softmax rows in a Numba-parallel kernel. It
then collapses the posterior-weighted probability residuals once per period and
reuses them across the 602 legacy utility coefficients, with independent score
columns reduced in parallel. The consumption and debt scores remain analytical.
Set `AUXILIARY_LEGACY_JACOBIAN=1` to use the retained reference implementation,
or pass `--numba-threads N` when running `model_em_algorithm.py` directly to
benchmark/control the Numba worker count. Neither option changes the sample or
the likelihood definition.

## Main estimation flow

`estimation_all_em.py` implements the Aguirregabiria--Mira / NPL loop:

1. `model_solution_em.simulate_all_states` creates the feasible state grids.
2. Optionally, `solve_many_continuations` calls `model_continuation_final` and
   `model_interpolate_terminal.build_interp_cache` builds the terminal-value
   interpolators.
3. Optionally, `model_em_algorithm.perform_em` estimates posterior weights for
   the eight auxiliary joint types. `model_predict_ccps` constructs the initial
   auxiliary CCPs. Downstream structural code continues to collapse/use only
   the schooling dimension until the resource types are deliberately added in
   a later step.
4. When `get_budget=True`, `model_getccp_sequence` converts the auxiliary CCPs
   into debt-indexed expected-value sequences. Then
   `model_fitloans_dynamic.estimate_budget_shock` fits the loan block and saves
   its parameters. `model_solution_em.reload_budgetshock_params` makes those
   parameters available to worker processes.
5. `model_solution_em.get_all_evt` solves the dynamic model backward for every
   invariant state and both education types.
6. `model_em_algorithm.prepare_vjt_feasible` maps solved values to observed
   person-period choices. `model_em_algorithm.likelihood` updates flow-utility
   parameters. The resulting CCPs feed the next NPL iteration.

The four switches near the top of `estimation_all_em.py` control expensive
prerequisites: `solve_continuation`, `solve_model`, `solve_qs`, and `get_budget`.
The default values reuse existing artifacts.

## Main simulation flow

`simulation_fit_conterfactual_em.py` is the driver. It uses
`model_solution_em` to solve the baseline/counterfactual value functions and
`model_simulation_em` to simulate choices and state transitions.
`model_simulation_nodebt.py` and `model_solution_nodebt.py` implement the
no-debt comparison. `simulation_fit_conterfactual_adjusted.py` is the older
post-choice debt adjustment exercise and is not the structural loan estimator.

## How the loan distribution is fitted now

The active file imported by the main estimator is `model_fitloans_dynamic.py`.
The other `model_fitloans_*` files are experiments or earlier variants:

- `model_fitloans_nosigma.py`: static/restricted variant.
- `model_fitloans_fast.py`: single-period optimized variant.
- `model_fitloans_dynamic_fast.py`: larger experimental dynamic variant.
- `model_fitloans_time.py` and `model_fitloans_ability.py`: alternative
  heterogeneity specifications.
- `model_fitdebt_new2.py` and files under `Old Codes`: legacy prototypes; do
  not import them into the main estimator.

For each period 1--9, the active fitter:

1. Loads observed invariant states, dynamic states, current debt, next debt,
   education choice, and income components from `model_em_algorithm`.
2. Keeps observations making an education choice and converts debt-grid indices
   to dollars.
3. Loads the type-specific CCP continuation sequence produced by
   `model_getccp_sequence`, then randomly draws a type using the EM posterior.
4. Constructs cash on hand before new borrowing as grants + transfers + labor
   income - accrued debt - tuition.
5. For every simulated budget shock, maximizes over the feasible debt grid:

       CRRA(cash + shock + new debt)
       + debt participation penalty
       + CCP-path continuation
       + discounted terminal value.

   The feasible grid respects the consumption floor, accrued current debt,
   annual borrowing limits, and lifetime borrowing limits.
6. Uses SMM to match, by parental-income quartile, mean positive debt, the share
   with debt, standard deviation of positive debt, and its 80th percentile. It
   also matches mean debt and the debt share by ability quartile. This gives 24
   moments per period.

The fitted parameters are:

- seven period-specific mean-shock coefficients (constant, three parental-
  income deviations, three ability deviations);
- one period-specific shock standard deviation;
- four parental-income-specific CRRA coefficients;
- a baseline debt participation penalty plus three parental-income deviations.

Outputs are `Estimates/risk_aversion.npy`,
`Estimates/budgetshock_params.npy`, and `Estimates/budgetshock_bestx.npy`.

## Important issues before the final estimation

1. **The current optimization is too large for Nelder--Mead.** With nine
   periods it searches over 80 parameters simultaneously, using a discontinuous
   debt-grid objective. This is the main computational reason convergence is
   difficult. Estimate in blocks (period shock location/scale first, common
   preference parameters second), then perform a short joint refinement.
2. **Identification is weak.** Shock means, risk aversion, and the debt penalty
   can all move average borrowing. A stable first specification should fix risk
   aversion from the consumption model, estimate period/group shock location
   and scale from loan moments, and only then test a parsimonious debt penalty.
3. **Use fixed simulation draws throughout an optimization.** The active code
   does this inside each outer iteration, but resamples between outer iterations.
   Compare objective values only on one permanent draw bank and use a separate
   draw bank for validation.
4. **Estimator/solver parameter consistency matters.** Schema version 2 records
   the debt penalty as baseline plus deviations. `model_solution_em` must apply
   exactly `[dp0, dp0+dp2, dp0+dp3, dp0+dp4]`; otherwise the estimated objective
   and solved model differ.
5. **Keep distribution experiments inside `budget_shock.py`.** The simulation
   now separates the log-wage shock from the additive dollar budget shock and
   uses the shared risk aversion and debt penalty. If the distribution family
   changes, update both `budget_shock.realization` (SMM and forward draws) and
   `budget_shock.quadrature` (backward integration) together.
6. **No completed loan estimate is currently present.** The existing
   `Model/Estimates` folder has EM and utility estimates but no
   `budgetshock_params.npy`; consequently a full loan-fit validation cannot yet
   be reported from saved results.

## Recommended final loan algorithm

Start with a deliberately identified two-stage SMM:

1. Fix CRRA and set the debt penalty to zero. For each period, estimate only the
   budget-shock mean (with parental-income and ability effects) and log standard
   deviation using common random numbers. Match debt participation, mean
   positive debt, and one upper quantile; reserve dispersion moments for
   validation or down-weight them when cells are small.
2. Pool periods and estimate at most four preference parameters (for example,
   one common CRRA and a baseline plus parental-income debt-penalty slope),
   holding the first-stage shock process fixed.
3. Jointly refine all parameters with a bounded optimizer suited to nonsmooth
   objectives (Powell or differential evolution for a small block), not an
   80-dimensional Nelder--Mead search.
4. Save estimates, reload them in `model_solution_em`, solve once, and validate
   with fresh shocks in `model_simulation_em`. Report fit by period, parental
   income, ability, and education sector, including cell sizes.

This staged routine can replace only `fit_budget_shock_multi`; the data loading,
debt-choice solver, saved-file interface, and hook in `estimation_all_em.py` can
remain in place.
